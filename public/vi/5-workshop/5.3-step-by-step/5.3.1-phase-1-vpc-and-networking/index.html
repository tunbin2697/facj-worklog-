5.3.1 Phase 1 - VPC và Networking

5.3.1 Phase 1 - VPC và Networking

Mục tiêu phase này là dựng nền tảng mạng cho toàn bộ stack MyFit: VPC, subnet, route table, security group.

Mapping ảnh -> bước

  • phase1 vpc and networking/create vpc -> tạo VPC
  • phase1 vpc and networking/create subnet -> tạo 4 subnet
  • phase1 vpc and networking/route table -> cấu hình route table
  • phase1 vpc and networking/security group -> tạo và nối các security group

Giá trị đã xác thực từ AWS CLI (stack hiện tại)

aws cloudformation list-stacks --stack-status-filter CREATE_COMPLETE --query "StackSummaries[?contains(StackName,'MyfitInfraStack')].[StackName,StackStatus]" --output table
aws ec2 describe-vpcs --vpc-ids vpc-0c93d1f17635865a7 --query "Vpcs[0].{VpcId:VpcId,Cidr:CidrBlock}" --output table
aws ec2 describe-subnets --filters Name=vpc-id,Values=vpc-0c93d1f17635865a7 --output json
aws ec2 describe-security-groups --group-ids sg-08002e12b1aa827d0 sg-04c05a2f2abab96cc sg-0ee7f30ea56f928c3 --output json
  • VPC: vpc-0c93d1f17635865a7, CIDR 10.0.0.0/16
  • DNS hostnames: true
  • DNS support: true
  • Internet Gateway: igw-041c82b7412211328
  • NAT Gateway trong VPC: không có ([])

5.3.1.1 Tạo VPC

  1. Vào VPC Console -> Create VPC.
  2. Chọn VPC only.
  3. Name tag: MyfitInfraStack/VPC.
  4. IPv4 CIDR: 10.0.0.0/16.
  5. Bật DNS hostnames và DNS resolution.
  6. Tạo VPC, sau đó tạo và attach Internet Gateway.

Ảnh tham chiếu:

5.3.1.2 Tạo 4 subnet trong 2 AZ

Tạo đúng 4 subnet sau để khớp stack hiện tại:

  1. Public subnet 1: 10.0.0.0/18 tại us-east-1a, auto-assign public IP = true.
  2. Public subnet 2: 10.0.64.0/18 tại us-east-1b, auto-assign public IP = true.
  3. Private isolated subnet 1: 10.0.128.0/18 tại us-east-1a, auto-assign public IP = false.
  4. Private isolated subnet 2: 10.0.192.0/18 tại us-east-1b, auto-assign public IP = false.

Ảnh tham chiếu:

5.3.1.3 Cấu hình route table

Theo stack hiện tại, không dùng NAT. Thiết kế route table là:

  1. Public route table cho subnet public có route 0.0.0.0/0 -> Internet Gateway.
  2. Isolated route table cho subnet private chỉ giữ route local 10.0.0.0/16.
  3. Không thêm default route internet cho subnet private.

Trong vài ảnh cũ có thể diễn đạt “private dùng default route table”. Dữ liệu AWS CLI của stack hiện tại cho thấy private subnets dùng route table riêng (rtb-0e6630e0556748c1b, rtb-0f7f5ad182142bd52) và không có route internet.

Ảnh tham chiếu:

5.3.1.4 Tạo 3 security group và rule liên thông

A. ALB Security Group

  • Name: MyfitInfraStack-BackendLBSecurityGroup...
  • Inbound: TCP 80 từ CloudFront prefix list pl-3b927c52
  • Outbound: TCP 8080 tới Backend Service SG

B. Backend Service Security Group

  • Name: MyfitInfraStack-BackendServiceSecurityGroup...
  • Inbound: TCP 8080 từ ALB SG
  • Outbound: all traffic (0.0.0.0/0)

C. Database Security Group

  • Name: MyfitInfraStack-DBSecurityGroup...
  • Inbound: TCP 5432 từ Backend Service SG
  • Outbound: all traffic (0.0.0.0/0)

Ảnh tham chiếu:

5.3.1.5 Checklist hoàn tất phase

  1. VPC ở trạng thái available, CIDR 10.0.0.0/16.
  2. Có 4 subnet đúng CIDR/AZ/public-ip flag như mục 5.3.1.2.
  3. Public subnets có route 0.0.0.0/0 -> igw-*.
  4. Private isolated subnets không có route internet.
  5. 3 security groups liên thông đúng luồng: CloudFront -> ALB -> ECS -> RDS.

Sau khi pass checklist này, bạn mới chuyển sang 5.3.2 (RDS) để tránh lỗi subnet group, SG reference hoặc lỗi connectivity.
